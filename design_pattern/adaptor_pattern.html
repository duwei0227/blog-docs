设计模式之适配器模式

概念

适配器模式是一种结构型设计模式，它允许接口不兼容的类能够一起工作。适配器模式就像现实生活中的电源适配器，可以让不同标准的插头连接到插座上。

核心思想：将一个类的接口转换成系统期望的另一个接口，使原本接口不兼容的类可以一起工作

✅ 适用场景：

  1. 整合第三方库：需要使用现有的类，但其接口不符合需求
  2. 系统升级兼容：新系统需要兼容旧系统的接口
  3. 统一多个类接口：多个类有相似功能但接口不同
  4. 复用遗留代码：老代码功能可用但接口不兼容

使用建议

  • 适配器模式应谨慎使用，特别是在详细设计阶段，它更多地用于解决现有系统的问题。
  • 在考虑修改一个正常运行的系统接口时，适配器模式是一个合适的选择。

🎯 典型例子：

  • 电压转换器：220V转110V
  • 读卡器：SD卡读卡器、TF卡读卡器
  • 日志框架适配：Log4j、SLF4J、Logback
  • 数据库驱动：JDBC驱动适配不同数据库
  • 媒体播放器：播放不同格式文件

类图

类图说明：

  • Client：客户端，使用Target接口
  • Target：目标接口，客户端期望的接口
  • Adaptee：被适配者，已有的但不兼容的类
  • Adapter：适配器，将Adaptee接口转换为Target接口
