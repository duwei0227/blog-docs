Rust 不依赖垃圾回收器，却能在编译期保证内存安全——秘密在于**所有权**（`ownership`）系统和**生命周期**（`lifetime`）机制。所有权决定每一块内存由谁持有、何时释放；借用（`borrow`）允许在不转移所有权的前提下访问数据；生命周期则是编译器用来验证引用始终有效的标注体系。理解这三者，是写出安全、高效 Rust 代码的核心前提。

## 一、所有权基础

### 1.1 三条所有权规则

Rust 的内存模型由三条不可违反的规则驱动：

| 规则 | 含义 |
|------|------|
| 每个值都有一个**所有者**（`owner`） | 值与变量绑定，变量即所有者 |
| 同一时刻只能有**一个所有者** | 不允许两个变量同时拥有同一块堆内存 |
| 所有者离开作用域，值被**自动丢弃**（`drop`） | 编译器在作用域末尾插入析构调用 |

### 1.2 移动语义（Move）

堆分配类型（如 `String`、`Vec`）在赋值或传参时发生**移动**（`move`）：原变量失效，所有权转移给新变量。栈上的元数据（指针 + 长度 + 容量）被浅拷贝，堆数据本身不复制。

**语法：**

```rust
let s2 = s1;          // s1 的所有权移动给 s2
fn f(s: String) { }   // 调用 f(s1) 后，s1 失效
```

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;           // s1 移动给 s2，s1 不再有效
    println!("s2 = {}", s2);
    // println!("{}", s1); // 编译错误：value borrowed here after move
}
```

运行结果：

```
s2 = hello
```

> 移动后使用已失效变量会在编译期报错，而非运行时崩溃——这正是 Rust 内存安全的核心保证。

### 1.3 Clone 与 Copy

当需要保留原始变量时，使用 `clone()` 进行深拷贝：

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone();   // 堆数据完整复制
    println!("s1 = {}, s2 = {}", s1, s2);
}
```

运行结果：

```
s1 = hello, s2 = hello
```

实现了 `Copy` `trait` 的类型（整数、浮点数、布尔、`char`、元素均为 `Copy` 的元组）在赋值时**自动复制**，原变量依然有效：

| 类型 | 是否 Copy |
|------|-----------|
| `i32`、`u64`、`f64`… | 是 |
| `bool`、`char` | 是 |
| `(i32, bool)` | 是（元素均为 `Copy`） |
| `String`、`Vec<T>` | 否（堆分配） |
| `&T`（不可变引用） | 是 |
| `&mut T`（可变引用） | 否 |

### 1.4 函数与所有权传递

函数调用遵循与赋值完全相同的语义：传入 `String` 会移动所有权，传入 `i32` 会复制：

```rust
fn takes_ownership(s: String) {
    println!("获得所有权: {}", s);
}

fn makes_copy(x: i32) {
    println!("复制整数: {}", x);
}

fn main() {
    let x = 5;
    let y = x;
    println!("x = {}, y = {}", x, y);   // Copy，两者均有效

    let s = String::from("world");
    takes_ownership(s);                   // s 的所有权移入函数
    // println!("{}", s);                 // 编译错误

    let n = 42;
    makes_copy(n);
    println!("n 依然有效: {}", n);        // Copy，n 仍可用
}
```

运行结果：

```
x = 5, y = 5
获得所有权: world
复制整数: 42
n 依然有效: 42
```

函数也可以通过返回值把所有权交还给调用者：

```rust
fn give_ownership() -> String {
    String::from("returned")
}

fn take_and_give_back(s: String) -> String {
    s
}

fn main() {
    let s1 = give_ownership();
    let s2 = String::from("hello");
    let s3 = take_and_give_back(s2);
    println!("s1 = {}, s3 = {}", s1, s3);
}
```

运行结果：

```
s1 = returned, s3 = hello
```

> 每次通过返回值传递所有权很繁琐。实际工程中，借用（`borrow`）才是首选——既能访问数据，又不转移所有权。

## 二、借用与引用

### 2.1 不可变借用

**语法：**

```rust
&T
```

**参数：**

| 符号 | 说明 |
|------|------|
| `&T` | 对类型 `T` 的不可变引用；不获取所有权，不能修改数据 |

借用通过 `&` 创建引用，函数结束后引用消失，数据所有权仍归原变量：

```rust
fn calculate_length(s: &String) -> usize {
    s.len()
}

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);
    println!("'{}' 的长度是 {}", s1, len);  // s1 仍然有效
}
```

运行结果：

```
'hello' 的长度是 5
```

### 2.2 可变借用

**语法：**

```rust
&mut T
```

**参数：**

| 符号 | 说明 |
|------|------|
| `&mut T` | 对类型 `T` 的可变引用；允许修改数据，原变量必须声明为 `mut` |

```rust
fn add_suffix(s: &mut String) {
    s.push_str(" world");
}

fn main() {
    let mut s = String::from("hello");
    add_suffix(&mut s);
    println!("{}", s);
}
```

运行结果：

```
hello world
```

### 2.3 借用规则汇总

Rust 的借用检查器（`borrow checker`）在编译期强制执行以下规则：

| 规则 | 说明 |
|------|------|
| 同一作用域内，不可变引用可以有**任意多个** | `&T` 之间不存在竞争 |
| 同一作用域内，可变引用**只能有一个** | 防止数据竞争（`data race`） |
| 不可变引用与可变引用**不能同时存在** | 读写互斥 |
| 引用必须始终有效 | 禁止悬垂引用（`dangling reference`） |

```rust
fn main() {
    let s = String::from("hello");
    let r1 = &s;
    let r2 = &s;    // 多个不可变借用合法
    println!("r1 = {}, r2 = {}", r1, r2);
    // r1、r2 作用域在此结束（NLL：非词法作用域生命周期）

    let r3 = &mut String::from("mutable");
    r3.push_str("!");
    println!("r3 = {}", r3);
}
```

运行结果：

```
r1 = hello, r2 = hello
r3 = mutable!
```

> `Rust 2018 Edition` 引入 `NLL`（Non-Lexical Lifetimes，非词法作用域生命周期），引用的有效范围缩短为最后一次使用处，而非词法作用域末尾，大幅减少借用冲突。

## 三、生命周期标注

### 3.1 悬垂引用：生命周期解决的问题

生命周期标注的核心目的是阻止**悬垂引用**（`dangling reference`）：引用指向的数据已被释放。下面的代码无法编译：

```rust
fn dangle() -> &String {       // 编译错误
    let s = String::from("hello");
    &s                         // s 在函数结束时被 drop，引用无效
}
```

编译器报错：`missing lifetime specifier`。正确做法是直接返回 `String` 转移所有权，而不是返回引用。

### 3.2 函数中的生命周期

#### 何时必须标注

只有当函数**返回引用**，且该引用与至少一个输入引用存在关联时，才需要手动标注生命周期。

| 函数签名特征 | 是否需要标注 | 原因 |
|-------------|-------------|------|
| 无引用返回值 | 否 | 编译器无需追踪返回引用的来源 |
| 单个引用参数 + 返回引用 | 否 | 省略规则二自动将输入生命周期赋给输出 |
| 多个引用参数 + 返回引用 | **是** | 编译器无法确定返回值绑定哪个参数 |
| 返回引用与所有参数无关 | 编译错误 | 必然是悬垂引用 |

#### 编译器的三步自动推断

在要求手动标注之前，编译器对函数签名中的所有引用按顺序应用三条**省略规则**（`elision rules`）。若三条规则全部应用后输出引用的生命周期仍未确定，才报错要求手动标注。

**规则一 — 为每个输入引用赋予独立生命周期**

每个引用参数各自获得一个全新生命周期参数。执行后输出引用的生命周期**仍未确定**，继续规则二：

```
fn f(x: &T)           →  fn f<'a>(x: &'a T)
fn f(x: &T, y: &U)   →  fn f<'a, 'b>(x: &'a T, y: &'b U)
```

**规则二 — 唯一输入生命周期传递给所有输出**

规则一执行后，若只剩**一个**输入生命周期参数，将它赋给所有输出引用；多个输入时跳过，进入规则三：

```
fn f<'a>(x: &'a T) -> &U   →   fn f<'a>(x: &'a T) -> &'a U   ✓
```

**规则三 — `&self` 的生命周期传递给所有输出**

若参数中有 `&self` 或 `&mut self`，将 `self` 的生命周期赋给所有输出引用：

```
fn f<'a, 'b>(&'a self, x: &'b T) -> &U   →   fn f<'a, 'b>(&'a self, x: &'b T) -> &'a U   ✓
```

三条规则执行后结果：

| 签名（省略形式） | 规则一 | 规则二 | 规则三 | 结果 |
|----------------|--------|--------|--------|------|
| `fn f(x: &T) -> &U` | `<'a>` | 唯一输入 → 输出得 `'a` | — | ✓ 自动推断 |
| `fn f(x: &T, y: &U) -> &V` | `<'a,'b>` | 两个输入，跳过 | 无 `self`，跳过 | ✗ 报错 |
| `fn f(&self) -> &T` | `<'a>` | 唯一输入 → 输出得 `'a` | — | ✓ 自动推断 |
| `fn f(&self, x: &T) -> &U` | `<'a,'b>` | 两个输入，跳过 | 输出得 `self` 的 `'a` | ✓ 自动推断 |

> 三条规则全部失败（如"多个输入引用 + 无 `self` + 有引用返回值"）时，编译器报 `missing lifetime specifier`，必须手动标注。3.4 节有每条规则的完整可运行示例。

#### 同一生命周期：`'a` 约束两个参数

**语法：**

```rust
fn func<'a>(x: &'a T, y: &'a T) -> &'a T
```

**参数：**

| 标注 | 说明 |
|------|------|
| `'a` | 生命周期参数名（惯例用单引号加小写字母）；描述约束关系，不改变实际存活时长 |
| `x: &'a T` | `x` 的引用生命周期至少为 `'a` |
| `y: &'a T` | `y` 的引用生命周期至少为 `'a` |
| `-> &'a T` | 返回引用的存活时间 ≤ `x` 与 `y` 中**较短**的那个 |

当 `x` 和 `y` 的实际生命周期不同时，编译器将 `'a` **收紧**（`constrain`）为两者的交集——即较短的那个：

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

fn main() {
    let s1 = String::from("long string");
    {
        let s2 = String::from("xyz");
        // 'a 被收紧为 s2 的生命周期（较短），result 必须在此作用域内使用
        let result = longest(s1.as_str(), s2.as_str());
        println!("最长的字符串: {}", result);
    }
    // 若在此处使用 result → 编译拒绝，因为 'a 不覆盖 s2 drop 之后
}
```

运行结果：

```
最长的字符串: long string
```

#### 不同生命周期：返回值只绑定其中一个参数

若返回值**只可能来自某一个参数**，可以为两个参数赋予独立的生命周期，让返回值只跟目标参数绑定，不受另一参数作用域约束：

**语法：**

```rust
fn func<'a, 'b>(x: &'a T, y: &'b T) -> &'a T
```

```rust
fn first<'a, 'b>(x: &'a str, _y: &'b str) -> &'a str {
    x   // 返回值只与 x 绑定，与 _y 无关
}

fn main() {
    let s1 = String::from("first");
    let result;
    {
        let s2 = String::from("second");
        result = first(s1.as_str(), s2.as_str());
    } // s2 在此 drop，但 result 生命周期绑定 s1，仍然有效
    println!("结果: {}", result);
}
```

运行结果：

```
结果: first
```

> 若把上例中的 `-> &'a str` 改成 `-> &'b str` 并返回 `_y`，则 `result` 的存活范围受 `s2` 限制，在内层作用域外使用会被编译器拒绝——这正是生命周期标注阻止悬垂引用的机制。

#### 参数引用规则汇总

| 场景 | 签名模板 | 典型用途 |
|------|----------|----------|
| 只读，不返回引用 | `fn f(x: &T)` | 打印、校验 |
| 只读，返回与输入相关的引用 | `fn f<'a>(x: &'a T) -> &'a U` | 字段提取、切片 |
| 多输入，返回其中之一 | `fn f<'a>(x: &'a T, y: &'a T) -> &'a T` | 比较、选择 |
| 多输入，返回值只绑定一个 | `fn f<'a, 'b>(x: &'a T, y: &'b U) -> &'a V` | 转换、格式化 |
| 可变借用 + 返回引用 | `fn f<'a>(x: &'a mut T) -> &'a U` | 就地修改后返回引用 |

### 3.3 结构体中的生命周期

结构体若持有引用字段，必须标注生命周期，声明"该结构体存活时间不超过所引用的数据"：

**语法：**

```rust
struct Name<'a> {
    field: &'a str,
}
```

```rust
#[derive(Debug)]
struct Excerpt<'a> {
    content: &'a str,
}

impl<'a> Excerpt<'a> {
    fn level(&self) -> i32 {
        3
    }

    fn announce(&self, announcement: &str) -> &str {
        println!("注意: {}", announcement);
        self.content
    }
}

fn main() {
    let novel = String::from("Call me Ishmael. Some years ago...");
    let first = novel.split('.').next().expect("找不到句号");
    let ex = Excerpt { content: first };
    println!("{:?}", ex);
    println!("等级: {}", ex.level());
    let part = ex.announce("重要段落");
    println!("内容: {}", part);
}
```

运行结果：

```
Excerpt { content: "Call me Ishmael" }
等级: 3
注意: 重要段落
内容: Call me Ishmael
```

> `impl` 块的生命周期标注 `impl<'a> Excerpt<'a>` 是必须的，因为 `Excerpt` 本身携带了生命周期参数。方法内部的省略规则（见 3.4）会进一步减少标注量。

### 3.4 生命周期省略规则

大多数场景下编译器能自动推断生命周期，无需显式标注。编译器按顺序应用三条**省略规则**（`elision rules`）。三条规则应用完毕后，若仍有输出引用的生命周期未确定，则报错——必须手动标注。

#### 规则一：为每个输入引用分配独立生命周期

每个引用参数各自获得一个全新的生命周期参数：

```
fn f(x: &T)            →  fn f<'a>(x: &'a T)
fn f(x: &T, y: &U)    →  fn f<'a, 'b>(x: &'a T, y: &'b U)
fn f(x: &T, y: &T)    →  fn f<'a, 'b>(x: &'a T, y: &'b T)
```

这是所有推断的起点。规则一执行后，输出引用的生命周期**仍然未知**，需要规则二或规则三来确定。

#### 规则二：唯一输入生命周期传递给所有输出

若规则一执行后只剩下**一个**输入生命周期参数，将它赋给所有输出引用：

```
fn f<'a>(x: &'a T) -> &U    →  fn f<'a>(x: &'a T) -> &'a U   ✓
```

规则一 + 规则二可以推断**单引用参数函数**的完整生命周期：

```rust
// 省略形式（用户写的）
fn first_word(s: &str) -> &str {
    s.split_whitespace().next().unwrap_or("")
}

// 编译器展开后（等价）
// fn first_word<'a>(s: &'a str) -> &'a str
```

若有**多个**输入引用参数，规则二无法适用（存在歧义），推断移入规则三。

#### 规则三：方法中 `&self` 的生命周期传递给所有输出

若参数列表中包含 `&self` 或 `&mut self`，将 `self` 的生命周期赋给所有输出引用：

```
fn f<'a, 'b>(&'a self, x: &'b T) -> &U    →  fn f<'a, 'b>(&'a self, x: &'b T) -> &'a U   ✓
```

这意味着方法的返回引用默认绑定 `self`，与其他参数的生命周期无关：

```rust
struct Config {
    host: String,
}

impl Config {
    // 省略形式
    fn host(&self) -> &str {
        &self.host
    }
    // 展开：fn host<'a>(&'a self) -> &'a str
    // 规则一：'a（self），规则三：输出绑定 'a ✓

    // 两个引用参数：规则三处理多参数情形
    fn pick<'b>(&self, _override: &'b str) -> &str {
        &self.host
    }
    // 展开：fn pick<'a, 'b>(&'a self, _override: &'b str) -> &'a str
    // 规则一：'a（self）、'b（_override），规则三：输出绑定 'a ✓
}

fn main() {
    let cfg = Config { host: String::from("localhost") };
    println!("host: {}", cfg.host());

    let result;
    {
        let ov = String::from("override-host");
        result = cfg.pick(&ov);
    } // ov drop，result 绑定 cfg（self），仍有效
    println!("pick: {}", result);
}
```

运行结果：

```
host: localhost
pick: localhost
```

#### 三条规则都无法解决时：必须手动标注

多个引用参数 + 无 `self` + 有引用返回值：三条规则应用完毕后输出生命周期仍未确定，编译器报错 `missing lifetime specifier`：

```rust
// 无法编译——编译器无法确定返回值绑定 x 还是 y
// fn longest(x: &str, y: &str) -> &str { ... }

// 必须手动声明：返回值与 x 和 y 的较短生命周期绑定
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

#### 三步推断全景表

| 省略前签名 | 规则一 | 规则二 | 规则三 | 结果 |
|-----------|--------|--------|--------|------|
| `fn f(x: &T) -> &U` | `fn f<'a>(x: &'a T) -> &U` | 输出获得 `'a` | 无 `self` | ✓ 自动推断 |
| `fn f(x: &T, y: &U) -> &V` | `fn f<'a,'b>(x: &'a T, y: &'b U) -> &V` | 两个输入，跳过 | 无 `self` | ✗ 报错，需手动标注 |
| `fn f(&self) -> &T` | `fn f<'a>(&'a self) -> &T` | 一个输入，`'a` 赋给输出 | — | ✓ 自动推断 |
| `fn f(&self, x: &T) -> &U` | `fn f<'a,'b>(&'a self, x: &'b T) -> &U` | 两个输入，跳过 | 输出获得 `'a` | ✓ 自动推断 |

## 四、`'static` 生命周期

`'static` 是最长的生命周期，表示引用在**整个程序运行期间**都有效。字符串字面量默认就是 `&'static str`，因为它们被编译进二进制的只读段：

**语法：**

```rust
let s: &'static str = "字面量";
fn f() -> &'static str { "常量" }
```

```rust
fn get_greeting() -> &'static str {
    "Hello, Rust!"
}

fn print_static(s: &'static str) {
    println!("静态字符串: {}", s);
}

fn main() {
    let s: &'static str = "程序运行期间始终有效";
    print_static(s);
    println!("{}", get_greeting());
}
```

运行结果：

```
静态字符串: 程序运行期间始终有效
Hello, Rust!
```

| 场景 | 是否为 `'static` |
|------|-----------------|
| 字符串字面量 `"abc"` | 是 |
| `Box::leak` 泄漏的堆数据 | 是 |
| `const` / `static` 变量 | 是 |
| 函数局部变量的引用 | 否 |
| 结构体字段引用（非 `'static` 标注） | 否 |

> 编译器报错提示"add `'static` bound"时，优先考虑是否真正需要静态生命周期，或者改用 `Arc<T>` / `String` 等拥有所有权的类型，而非盲目添加 `'static`。

### 4.1 何时应该使用 `'static`

`'static` 主要有两种含义：

1. `&'static T`：这个引用指向的数据在整个程序运行期间都有效。
2. `T: 'static`：类型 `T` 内部不包含非静态借用，或者只包含 `'static` 借用；它并不表示这个值本身一定存活到程序结束。

主动写 `&'static T` 的常见场景很少，通常只在下面这些地方出现：

```rust
const APP_NAME: &str = "blog-docs";
static VERSION: &str = "1.0.0";

fn app_name() -> &'static str {
    APP_NAME
}
```

- 返回字符串字面量、全局常量、全局静态配置等确实随程序存在的数据。
- 定义全局只读数据、缓存键、错误码、命令名称等不会动态释放的引用。
- 少数场景用 `Box::leak` 将堆数据泄漏为 `'static` 引用，例如一次性初始化后全局复用的数据。这个做法会故意跳过释放，应谨慎使用。

更常见的是 `T: 'static` 约束，尤其在线程、异步任务、回调注册等场景中：

```rust
use std::thread;

fn main() {
    let name = String::from("Rust");

    thread::spawn(move || {
        println!("{}", name);
    });
}
```

`thread::spawn` 要求闭包满足 `'static`，因为新线程可能比当前函数活得更久。这里通过 `move` 把 `String` 的所有权移入线程，闭包内部不再借用局部变量，因此可以满足 `T: 'static`。

### 4.2 是否需要主动使用 `'static`

大多数时候不需要。Rust 的生命周期应尽量由编译器推断，只有当 API 边界确实要求长期持有数据时，才考虑 `'static`。

不要把 `'static` 当作解决生命周期报错的默认手段：

```rust
fn wrong() -> &'static str {
    let s = String::from("temporary");
    &s // 错误：s 在函数结束时被释放，不能返回 'static 引用
}
```

如果函数需要返回动态生成的数据，应返回拥有所有权的类型：

```rust
fn right() -> String {
    String::from("temporary")
}
```

如果异步任务、线程或回调需要长期使用数据，优先移动所有权或使用共享所有权：

```rust
use std::sync::Arc;
use std::thread;

fn main() {
    let config = Arc::new(String::from("config"));
    let cloned = Arc::clone(&config);

    thread::spawn(move || {
        println!("{}", cloned);
    });
}
```

实战判断顺序：

| 问题 | 推荐做法 |
|------|----------|
| 数据本来就是字符串字面量或全局静态数据 | 可以使用 `&'static str` / `&'static T` |
| 函数返回动态创建的数据 | 返回 `String` / `Vec<T>` / 自定义拥有所有权的类型 |
| 线程、异步任务、回调要求 `T: 'static` | 用 `move` 转移所有权，或用 `Arc<T>` 共享所有权 |
| 只是为了消除生命周期报错 | 不要优先加 `'static`，先检查返回值是否借用了局部变量 |
| 结构体需要长期保存外部数据 | 优先保存拥有所有权的字段，如 `String`，而不是 `&'static str` |

## 五、泛型与生命周期组合

生命周期参数可以与泛型类型参数同时使用。两者都写在尖括号中，生命周期参数在前：

**语法：**

```rust
fn func<'a, T: Trait>(x: &'a str, y: &'a str, extra: T) -> &'a str
```

```rust
use std::fmt::Display;

fn longest_with_announcement<'a, T>(
    x: &'a str,
    y: &'a str,
    ann: T,
) -> &'a str
where
    T: Display,
{
    println!("公告: {}", ann);
    if x.len() > y.len() { x } else { y }
}

fn main() {
    let s1 = String::from("long string is long");
    let s2 = String::from("xyz");
    let result = longest_with_announcement(&s1, &s2, "今天天气很好");
    println!("最长: {}", result);
}
```

运行结果：

```
公告: 今天天气很好
最长: long string is long
```

## 六、常见错误模式速查

| 错误信息（关键词） | 原因 | 修复思路 |
|-------------------|------|----------|
| `value borrowed here after move` | 移动后使用原变量 | 改用 `clone()` 或改传引用 |
| `cannot borrow as mutable because it is not declared as mutable` | 未声明 `mut` | 变量声明加 `mut` |
| `cannot borrow as mutable more than once` | 同时存在多个 `&mut` | 缩短第一个可变借用的作用域 |
| `cannot borrow as immutable because it is also borrowed as mutable` | 可变借用与不可变借用共存 | 调整借用顺序或作用域 |
| `missing lifetime specifier` | 函数/结构体返回引用但无生命周期标注 | 添加 `<'a>` 标注并标注参数与返回值 |
| `returns a reference to data owned by the current function` | 试图返回局部变量的引用 | 返回拥有所有权的类型（`String`、`Vec`…） |
| `lifetime may not live long enough` | 引用的存活范围短于使用范围 | 调整生命周期约束或改用拥有所有权的类型 |
