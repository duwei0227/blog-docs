枚举（`enum`）和模式匹配（`pattern matching`）是 Rust 表达“有限状态”和“分支处理”的核心工具。`enum` 让一个类型可以拥有多个互斥变体，`match` 则要求你把这些变体处理完整。两者配合后，很多原本依赖空值、错误码、类型标记字段的逻辑，都可以交给编译器检查。

## 一、枚举基础

### 1.1 为什么需要枚举

当一个值只可能处于几个明确状态之一时，枚举比字符串、整数常量或多个布尔字段更安全。

例如 HTTP 请求方法只有有限几种：

```rust
enum HttpMethod {
    Get,
    Post,
    Put,
    Delete,
}
```

`HttpMethod` 是一个类型，`Get`、`Post`、`Put`、`Delete` 是它的变体（variant）。同一时刻，一个 `HttpMethod` 值只能是其中一个变体。

```rust
#[derive(Debug)]
enum HttpMethod {
    Get,
    Post,
    Put,
    Delete,
}

fn main() {
    let method = HttpMethod::Post;
    println!("method = {:?}", method);
}
```

运行结果：

```text
method = Post
```

> 注意：变体通过 `EnumName::VariantName` 访问。枚举变体不是字符串，也不是整数常量，而是受类型系统检查的值。

### 1.2 枚举比常量更适合表达状态

如果用字符串表达状态，拼写错误只能在运行时暴露：

```rust
fn handle_status(status: &str) {
    if status == "success" {
        println!("成功");
    } else if status == "failed" {
        println!("失败");
    } else {
        println!("未知状态: {status}");
    }
}
```

使用枚举后，状态集合由类型定义，不能随意写出一个不存在的状态：

```rust
enum TaskStatus {
    Pending,
    Running,
    Success,
    Failed,
}
```

| 表达方式 | 优点 | 缺点 |
|----------|------|------|
| 字符串 | 简单、适合外部输入 | 容易拼错，分支无法穷尽检查 |
| 整数常量 | 存储紧凑 | 可读性差，非法值难以避免 |
| 多个布尔字段 | 写法直观 | 容易出现互斥状态同时为真的问题 |
| 枚举 | 类型安全、状态互斥、可穷尽匹配 | 需要先建模变体 |

实际项目中，内部业务状态优先使用枚举；只有在读取配置、HTTP 参数、数据库字段时，才先接收字符串或数字，再转换为枚举。

## 二、带数据的枚举变体

枚举变体不仅能表示“是哪一种”，还可以携带不同结构的数据。这是 Rust 枚举比很多语言中的普通枚举更强的地方。

### 2.1 每个变体可以携带不同数据

假设消息系统中有三类消息：文本、图片、退出信号。它们需要的数据并不相同：

```rust
#[derive(Debug)]
enum Message {
    Text(String),
    Image { url: String, width: u32, height: u32 },
    Quit,
}

fn main() {
    let text = Message::Text(String::from("hello"));
    let image = Message::Image {
        url: String::from("https://example.com/a.png"),
        width: 640,
        height: 480,
    };
    let quit = Message::Quit;

    println!("{:?}", text);
    println!("{:?}", image);
    println!("{:?}", quit);
}
```

运行结果：

```text
Text("hello")
Image { url: "https://example.com/a.png", width: 640, height: 480 }
Quit
```

同一个 `enum` 中可以混合使用三种变体形式：

| 变体形式 | 示例 | 适合场景 |
|----------|------|----------|
| 单元变体 | `Quit` | 只表达状态，不需要额外数据 |
| 元组变体 | `Text(String)` | 数据字段较少，字段含义明显 |
| 结构体变体 | `Image { url, width, height }` | 字段较多，需要名字增强可读性 |

### 2.2 枚举可以定义方法

和 `struct` 一样，`enum` 也可以通过 `impl` 块定义方法：

```rust
enum Message {
    Text(String),
    Image { url: String, width: u32, height: u32 },
    Quit,
}

impl Message {
    fn name(&self) -> &'static str {
        match self {
            Message::Text(_) => "text",
            Message::Image { .. } => "image",
            Message::Quit => "quit",
        }
    }
}

fn main() {
    let msg = Message::Text(String::from("hello"));
    println!("消息类型: {}", msg.name());
}
```

运行结果：

```text
消息类型: text
```

这里的 `name(&self)` 只读取消息类型，不需要取得所有权，所以接收者使用 `&self`。`match self` 时，匹配的是 `&Message`，因此分支中的数据默认也是借用方式处理。

## 三、`Option<T>`：用枚举替代空值

Rust 标准库没有传统意义上的 `null`。当一个值可能不存在时，通常使用 `Option<T>`：

```rust
enum Option<T> {
    None,
    Some(T),
}
```

实际代码不需要自己定义 `Option<T>`，它已经在标准库预导入范围内，可以直接使用 `Some` 和 `None`。

### 3.1 基础用法

```rust
fn find_user_name(id: u32) -> Option<String> {
    if id == 1 {
        Some(String::from("Alice"))
    } else {
        None
    }
}

fn main() {
    let name = find_user_name(1);
    println!("查询结果: {:?}", name);

    let missing = find_user_name(2);
    println!("查询结果: {:?}", missing);
}
```

运行结果：

```text
查询结果: Some("Alice")
查询结果: None
```

`Option<String>` 的含义是：要么有一个 `String`，要么没有值。调用者必须显式处理这两种情况，不能像可空引用那样直接使用内部值。

### 3.2 `Option<T>` 不能直接当作 `T` 使用

下面的代码不能直接运行：

```rust
// 不能直接运行：Option<i32> 和 i32 是不同类型
fn main() {
    let x: Option<i32> = Some(5);
    let y = x + 1;
    println!("{y}");
}
```

原因是 `x` 可能是 `None`。如果编译器允许直接加法，那么 `None + 1` 的行为就会变得不明确。

正确做法是先把 `Some` 和 `None` 拆开：

```rust
fn add_one(value: Option<i32>) -> Option<i32> {
    match value {
        Some(n) => Some(n + 1),
        None => None,
    }
}

fn main() {
    println!("{:?}", add_one(Some(5)));
    println!("{:?}", add_one(None));
}
```

运行结果：

```text
Some(6)
None
```

## 四、`match` 表达式

`match` 是 Rust 最重要的模式匹配语法。它根据值的形状选择分支，并且要求所有可能情况都被覆盖。

### 4.1 基础语法

**语法：**

```rust
match value {
    pattern1 => expression1,
    pattern2 => expression2,
    pattern3 => {
        statement;
        expression
    }
}
```

`match` 是表达式，每个分支都要产生同一种返回类型。

```rust
enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

// 所有权会发生 move 
fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}

fn main() {
    let coin = Coin::Dime;
    println!("面值: {} cents", value_in_cents(coin));
}
```

运行结果：

```text
面值: 10 cents
```

### 4.2 分支可以执行多行代码

当分支逻辑不止一个表达式时，使用代码块：

```rust
enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => {
            println!("这是一枚小面值硬币");
            1
        }
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}

fn main() {
    let value = value_in_cents(Coin::Penny);
    println!("面值: {value}");
}
```

运行结果：

```text
这是一枚小面值硬币
面值: 1
```

注意代码块最后一行没有分号，因为它是该分支的返回值。

### 4.3 `match` 必须穷尽所有情况

如果遗漏某个变体，编译器会报错：

```rust
// 不能直接运行：遗漏了 Coin::Quarter
enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
    }
}
```

`match` 的穷尽性检查能避免“新增状态后忘记改分支”的问题。对于业务状态、协议类型、错误分类，这个特性非常有价值。

### 4.4 `_` 通配符和兜底分支

如果只关心部分情况，可以使用 `_` 匹配剩余所有值：

```rust
fn describe_number(n: u8) -> &'static str {
    match n {
        0 => "zero",
        1 => "one",
        _ => "many",
    }
}

fn main() {
    println!("{}", describe_number(0));
    println!("{}", describe_number(7));
}
```

运行结果：

```text
zero
many
```

> 注意：`_` 很方便，但也会吞掉未来新增的情况。对业务枚举做重构时，如果希望编译器提醒所有需要修改的位置，应尽量显式列出每个变体。

## 五、在模式中绑定数据

模式匹配不仅能判断变体，还能把变体中的数据绑定到变量。

### 5.1 匹配枚举中的字段

```rust
enum Message {
    Text(String),
    Image { url: String, width: u32, height: u32 },
    Quit,
}

fn handle_message(msg: Message) {
    match msg {
        Message::Text(content) => {
            println!("文本消息: {content}");
        }
        Message::Image { url, width, height } => {
            println!("图片消息: {url}, 尺寸: {width}x{height}");
        }
        Message::Quit => {
            println!("退出");
        }
    }
}

fn main() {
    handle_message(Message::Text(String::from("hello")));
    handle_message(Message::Image {
        url: String::from("/tmp/a.png"),
        width: 320,
        height: 200,
    });
}
```

运行结果：

```text
文本消息: hello
图片消息: /tmp/a.png, 尺寸: 320x200
```

在 `Message::Text(content)` 中，`content` 会取得内部 `String` 的所有权。因为 `handle_message` 本身接收 `msg: Message`，所以移动内部数据是合理的。

### 5.2 借用匹配，避免移动数据

如果匹配后还想继续使用原值，应匹配引用，或在模式中使用 `ref`。

```rust
#[derive(Debug)]
enum Message {
    Text(String),
    Quit,
}

fn print_message(msg: &Message) {
    match msg {
        Message::Text(content) => println!("文本: {content}"),
        Message::Quit => println!("退出"),
    }
}

fn main() {
    let msg = Message::Text(String::from("hello"));
    print_message(&msg);
    println!("原消息仍可使用: {:?}", msg);
}
```

运行结果：

```text
文本: hello
原消息仍可使用: Text("hello")
```

`print_message` 接收 `&Message`，所以 `content` 的类型是 `&String`，不会移动内部字符串。

### 5.3 使用 `..` 忽略不关心的字段

结构体变体字段很多时，可以只匹配关心的字段：

```rust
enum Event {
    MouseClick { x: i32, y: i32, button: String },
    KeyPress { key: char },
}

fn main() {
    let event = Event::MouseClick {
        x: 120,
        y: 80,
        button: String::from("left"),
    };

    match event {
        Event::MouseClick { x, y, .. } => {
            println!("点击坐标: ({x}, {y})");
        }
        Event::KeyPress { key } => {
            println!("按键: {key}");
        }
    }
}
```

运行结果：

```text
点击坐标: (120, 80)
```

`..` 表示忽略剩余字段，适合字段较多但分支只需要部分信息的场景。

对于结构体变体，`..` 不要求只能忽略尾部字段。只要字段有名字，就可以直接写出任意关心的字段，其余字段统一交给 `..`：

```rust
enum Message {
    Data { a: i32, b: i32, c: i32, d: i32 },
}

fn main() {
    let msg = Message::Data {
        a: 1,
        b: 2,
        c: 3,
        d: 4,
    };

    match msg {
        Message::Data { a, c, .. } => {
            println!("a = {a}, c = {c}");
        }
    }
}
```

运行结果：

```text
a = 1, c = 3
```

这里的 `Message::Data { a, c, .. }` 只绑定 `a` 和 `c`，`b`、`d` 都会被忽略。因为字段是具名字段，匹配时不依赖字段声明顺序。

如果是元组变体，字段没有名字，只能按位置匹配。此时想取第 1 个和第 3 个字段，通常使用 `_` 忽略中间和末尾字段：

```rust
enum Message {
    Data(i32, i32, i32, i32),
}

fn main() {
    let msg = Message::Data(1, 2, 3, 4);

    match msg {
        Message::Data(a, _, c, _) => {
            println!("a = {a}, c = {c}");
        }
    }
}
```

运行结果：

```text
a = 1, c = 3
```

> **说明**：`..` 在同一个模式中只能出现一次，适合忽略一段连续的剩余内容；如果要在元组变体中跳过多个不连续位置，使用 `_` 更清晰。

## 六、`if let` 与 `while let`

当只关心一个模式，其他情况直接忽略时，`match` 会显得啰嗦。此时可以使用 `if let`。

### 6.1 `if let` 简化单分支匹配

```rust
fn main() {
    let value = Some(3);

    if let Some(n) = value {
        println!("取到值: {n}");
    }
}
```

运行结果：

```text
取到值: 3
```

上面代码等价于：

```rust
fn main() {
    let value = Some(3);

    match value {
        Some(n) => println!("取到值: {n}"),
        None => (),
    }
}
```

选择规则很简单：

| 场景 | 推荐写法 |
|------|----------|
| 必须处理所有分支 | `match` |
| 只关心一个分支 | `if let` |
| 只关心一个分支，但还要处理其他情况 | `if let ... else` |
| 循环中持续取出某种模式 | `while let` |

### 6.2 `if let ... else`

```rust
fn parse_port(input: Option<u16>) {
    if let Some(port) = input {
        println!("端口: {port}");
    } else {
        println!("未提供端口，使用默认值");
    }
}

fn main() {
    parse_port(Some(8080));
    parse_port(None);
}
```

运行结果：

```text
端口: 8080
未提供端口，使用默认值
```

如果 `else` 分支也变复杂，或者后续可能增加更多情况，应改回 `match`，可读性会更好。

### 6.3 `while let` 处理循环匹配

`while let` 常用于持续从栈、队列、迭代器中取值：

```rust
fn main() {
    let mut stack = vec![1, 2, 3];

    while let Some(value) = stack.pop() {
        println!("弹出: {value}");
    }
}
```

运行结果：

```text
弹出: 3
弹出: 2
弹出: 1
```

`Vec::pop()` 返回 `Option<T>`：有元素时返回 `Some(value)`，空数组时返回 `None`，循环结束。

## 七、常见模式写法

模式匹配并不只用于枚举，也可以用于字面量、范围、元组、结构体、引用等形状。

### 7.1 多个模式使用 `|`

```rust
fn is_weekend(day: &str) -> bool {
    match day {
        "Saturday" | "Sunday" => true,
        _ => false,
    }
}

fn main() {
    println!("{}", is_weekend("Saturday"));
    println!("{}", is_weekend("Monday"));
}
```

运行结果：

```text
true
false
```

### 7.2 范围模式

```rust
fn grade(score: u8) -> &'static str {
    match score {
        90..=100 => "A",
        80..=89 => "B",
        60..=79 => "C",
        _ => "D",
    }
}

fn main() {
    println!("{}", grade(95));
    println!("{}", grade(72));
}
```

运行结果：

```text
A
C
```

`..=` 是闭区间范围模式，包含两端的值。

### 7.3 匹配守卫

匹配守卫（`match guard`）是在模式后增加 `if` 条件：

```rust
fn describe(value: Option<i32>) -> &'static str {
    match value {
        Some(n) if n > 0 => "positive",
        Some(0) => "zero",
        Some(_) => "negative",
        None => "none",
    }
}

fn main() {
    println!("{}", describe(Some(10)));
    println!("{}", describe(Some(-3)));
    println!("{}", describe(None));
}
```

运行结果：

```text
positive
negative
none
```

匹配守卫适合表达“结构匹配成功后，还要满足额外条件”的分支。但守卫太多时，说明逻辑可能需要拆成函数。

### 7.4 `@` 绑定匹配到的值

`@` 可以在判断范围的同时，把匹配到的值绑定到变量：

```rust
fn main() {
    let age = 18;

    match age {
        n @ 0..=17 => println!("未成年人: {n}"),
        n @ 18..=59 => println!("成年人: {n}"),
        n => println!("老年人: {n}"),
    }
}
```

运行结果：

```text
成年人: 18
```

## 八、实战示例：请求解析结果建模

下面用枚举建模一个简单的请求解析流程。目标是区分“解析成功”“缺少字段”“字段格式错误”三种情况。

### 8.1 定义结果枚举

```rust
#[derive(Debug)]
struct Request {
    user_id: u64,
    action: String,
}

#[derive(Debug)]
enum ParseResult {
    Ok(Request),
    MissingField(&'static str),
    InvalidField { field: &'static str, reason: String },
}

fn parse_request(user_id: Option<&str>, action: Option<&str>) -> ParseResult {
    let Some(user_id) = user_id else {
        return ParseResult::MissingField("user_id");
    };

    let Some(action) = action else {
        return ParseResult::MissingField("action");
    };

    let user_id = match user_id.parse::<u64>() {
        Ok(id) => id,
        Err(err) => {
            return ParseResult::InvalidField {
                field: "user_id",
                reason: err.to_string(),
            };
        }
    };

    ParseResult::Ok(Request {
        user_id,
        action: action.to_string(),
    })
}

fn print_result(result: ParseResult) {
    match result {
        ParseResult::Ok(request) => {
            println!("解析成功: user_id={}, action={}", request.user_id, request.action);
        }
        ParseResult::MissingField(field) => {
            println!("缺少字段: {field}");
        }
        ParseResult::InvalidField { field, reason } => {
            println!("字段格式错误: {field}, 原因: {reason}");
        }
    }
}

fn main() {
    print_result(parse_request(Some("42"), Some("login")));
    print_result(parse_request(None, Some("login")));
    print_result(parse_request(Some("abc"), Some("login")));
}
```

运行结果：

```text
解析成功: user_id=42, action=login
缺少字段: user_id
字段格式错误: user_id, 原因: invalid digit found in string
```

这个例子中有几个设计点：

| 设计 | 原因 |
|------|------|
| `ParseResult::Ok(Request)` | 成功时携带完整解析后的请求对象 |
| `MissingField(&'static str)` | 缺少字段只需要字段名，字符串字面量即可 |
| `InvalidField { field, reason }` | 字段错误需要字段名和具体原因，结构体变体更清晰 |
| `let Some(x) = ... else` | 先处理失败路径，让主流程保持靠左 |
| `match result` | 输出时必须穷尽所有解析结果 |

> **提示**：真实项目中，解析失败通常会使用 `Result<T, E>`。当错误类型有多个清晰分类时，`E` 本身经常就是一个枚举。

## 九、常见陷阱与最佳实践

### 9.1 不要滥用 `_` 掩盖业务变体

```rust
enum OrderStatus {
    Created,
    Paid,
    Shipped,
    Canceled,
}

fn can_cancel(status: OrderStatus) -> bool {
    match status {
        OrderStatus::Created | OrderStatus::Paid => true,
        _ => false,
    }
}
```

这段代码可以工作，但如果未来新增 `Refunding`、`PaymentFailed` 等状态，`_` 会默认返回 `false`，编译器不会提醒你重新判断业务规则。

更适合业务状态的写法是显式列出：

```rust
fn can_cancel(status: OrderStatus) -> bool {
    match status {
        OrderStatus::Created => true,
        OrderStatus::Paid => true,
        OrderStatus::Shipped => false,
        OrderStatus::Canceled => false,
    }
}
```

### 9.2 注意匹配时的所有权移动

匹配拥有所有权的值时，非 `Copy` 字段可能被移动：

```rust
// 不能直接运行：content 被移动后，msg 不能再整体使用
#[derive(Debug)]
enum Message {
    Text(String),
    Quit,
}

fn main() {
    let msg = Message::Text(String::from("hello"));

    match msg {
        Message::Text(content) => println!("{content}"),
        Message::Quit => println!("quit"),
    }

    println!("{:?}", msg);
}
```

如果匹配后仍要使用原值，应匹配引用：

```rust
#[derive(Debug)]
enum Message {
    Text(String),
    Quit,
}

fn main() {
    let msg = Message::Text(String::from("hello"));

    match &msg {
        Message::Text(content) => println!("{content}"),
        Message::Quit => println!("quit"),
    }

    println!("{:?}", msg);
}
```

运行结果：

```text
hello
Text("hello")
```

### 9.3 `if let` 不会做穷尽性提醒

`if let` 只处理你写出的模式，其他情况会被忽略：

```rust
fn main() {
    let value: Option<i32> = None;

    if let Some(n) = value {
        println!("值: {n}");
    }
}
```

这段代码在 `None` 时什么都不做。如果 `None` 代表重要业务分支，应使用 `match` 或 `if let ... else`。

### 9.4 枚举命名要表达领域含义

不要把所有枚举都命名成 `Type`、`Kind`、`Status`，除非上下文已经很明确。更好的命名应体现领域：

| 不推荐 | 推荐 |
|--------|------|
| `Status` | `OrderStatus` |
| `Kind` | `MessageKind` |
| `Type` | `PaymentMethod` |
| `ResultType` | `ParseResult` |

清晰命名能让 `match` 分支自解释，减少额外注释。

## 十、速查表

### 10.1 枚举与匹配语法

| 语法                          | 说明                            |
| --------------------------- | ----------------------------- |
| `enum E { A, B }`           | 定义枚举                          |
| `E::A`                      | 构造枚举变体                        |
| `A(T)`                      | 元组变体，携带位置字段                   |
| `A { x: T }`                | 结构体变体，携带命名字段                  |
| `match value { ... }`       | 穷尽匹配                          |
| `_`                         | 匹配任意值但不绑定                     |
| `..`                        | 忽略剩余字段；具名字段可写成 `{ a, c, .. }` |
| `a..=b`                     | 闭区间范围模式                       |
| `pattern if condition`      | 匹配守卫                          |
| `name @ pattern`            | 匹配同时绑定值                       |
| `if let pattern = value`    | 单模式匹配                         |
| `while let pattern = value` | 循环单模式匹配                       |

### 10.2 选择建议

| 需求 | 推荐方式 |
|------|----------|
| 表达有限互斥状态 | `enum` |
| 某些状态需要携带不同数据 | 带数据的枚举变体 |
| 值可能不存在 | `Option<T>` |
| 操作可能成功或失败 | `Result<T, E>`，其中 `E` 可用枚举建模 |
| 必须覆盖所有情况 | `match` |
| 只关心一个成功分支 | `if let` |
| 循环取出 `Option` 中的值 | `while let` |
| 匹配后还要继续使用原值 | 匹配引用，例如 `match &value` |

## 总结

枚举用于建模“一个值可能是几种形态之一”，模式匹配用于安全地拆开这些形态。Rust 把二者和类型系统结合起来，使状态遗漏、空值误用、错误分支遗漏这类问题尽量在编译期暴露。

实际开发中可以遵循几个简单原则：

1. 内部业务状态优先用 `enum`，不要长期依赖字符串或整数常量。
2. 值可能不存在时使用 `Option<T>`，不要用特殊值模拟空值。
3. 分支必须完整处理时使用 `match`，只关心单个分支时使用 `if let`。
4. 业务枚举尽量显式列出变体，谨慎使用 `_`。
5. 匹配包含 `String`、`Vec<T>` 等非 `Copy` 数据的枚举时，先判断是否需要借用匹配。

掌握枚举和模式匹配后，Rust 代码会更像是在描述业务状态机：每种状态是什么、携带什么数据、每个分支如何处理，都由类型系统和编译器一起约束。
